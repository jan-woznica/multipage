HTML VERZE

Každý soubor na začátku musí obsahovat informaci, o jaký dokument se jedná. V jakém jazyku na soubor napsán

 <!DOCTYPE html>
<html lang="cs">

Doctype dříve byl odkaz na souhrn pravidel, ale nyní je to již pouze artefakt, který se musí zahrnout, aby vše fungovalo
Vždy nastavit jazyk, SEO bude lépe indexovat stránku, jazyk lze nastavit i pomocí inline řádků. Když nebudu vědět zkratky jazyků tak vyhledat language tags

Soubor ukládat jako index.html. Pro kontrolu dokumentů využít Markup Validation Service a při testu webů stačí dva prohlížeče (Firefox, Chrome, Opera nebo Edge
Pro nastavení https lze využít Lets encrypt, který obsahuje nástroje a instrukce jak automaticky vytvořit a instalovat potřebné certifikáty

Meta jsou metadata (informace) pro stránku a existuje několik druhů
Charset je potřeba nadefinovat co nejdříve, jelikož slouží ke kódování stránky a UTF používá téměř všechny znaky na světě
Další Charsety: Ascii (první kódovací styl), ANSI 1252 (windows original), ISO-8859-1 (pro HTML 4)

<meta name="viewport" content="width=device-width, initial-scale=1.0">
<meta name="description" content="HTML základy">
<meta name="keywords" content="HTML, základy">
<meta name="author" content="Jan Woznica"> 
<title>HTML BASICS<title>

viewport optimalizuje stránku pro mobily, description je text, který se zobrazuje pod názvem stránky při vyhledávání (pro přidání i podkapitol slouží google webmaster tools)
Specifikace description obsahuje klíčové fráze, aby se stránka zobrazila výše ve vyhledávání SEO
Další metadata: open graph (protokol, který vytvořil Facebook, aby se stránka zobrazila s obrázkem i popisem)
Author odkazuje na autora stránky a title nastaví název stránce

<link rel="icon" href="favicon.ico" type="image/x-icon" />

Ikony lze vidět na záložce stránky, buď lze nastavit obrázek nebo třeba vlastní favicon (krátký výraz pro favorites icon) jsou 16 pixelové
favicon.ico je soubor s iconou v aktuální složce, ale i stránka, kde si lze vytvářet vlastní favicony
Pokud stránka využívá Content Security Policy (CSP) a budu mít problém s nezobrazováním ikony, tak zkontrolovat Content-Security-Policy
Lze odkazovat na css soubory tkzv stylesheet nebo JavaScript, tyto soubory jsou buď ve složce nebo na internetu

IFRAME, EMBED A OBJECT

V době pluginů (Flash, Java Applets), které umožňovali programátorům vložit větší obsah, který nebyl dostupný za pomoci html pouze, zapojení těchto technologií bylo provedeno za pomocí object a embed elementů, ale ty se dneš již nepoužívají
Iframe se přestal také používat, protože byl častým terčem hackerů pomocí clickjackingu (vložení neviditelného iframu do dokumentu nebo vložení mé stránky do jejich k zachycení uživatelských informací
Jiné stránky nelze vkládat, protože mají nastaven csp derictiva a zároveň autor má autorská práva, ale i tak se doporučuje používat https zabezpečení
pomocí iframe lze vložit stránky, videa z youtube nebo mapy, které budou vloženy formou framu v okně stránky atribut sandbox slouží, aby ifrmay neměli změny v html, konfigurace CSP directive(content security policy) zprostředkuje set http nadpisů (metadata poslána spolu se stránkou a lze tedy nastavit x-Frame options, který zakazuje vkládání mého html do cizího html 

    <iframe src="url" style="border:none; height:400px; width:600px;" name="iframe_a" title="most" allowFullscreen></iframe>

Následující tlačítko vloží jiný link do iframu

Vložit jiný link

<a href="url" target="iframe_a">text</a></p>

<scope> se přidává do th, aby řekl screanreaderům na co je header, úplně nejjednodušší možnost je <scope="col"td označuje buňky data tabulky

<rowspan> na více řádků a <colspan> na sloupce
<colgroup> slouží ke stylování pro celý sloupec a <span> říká počet slupců, lze nastavit border, background, width a visibility

Sloupci lze udělit thead,tbody,tfoot jako v layoutu a je to vhodné pro stylování tabulek a tabulky lze i nestovat (vložit jednu tabulku do buňky druhé
